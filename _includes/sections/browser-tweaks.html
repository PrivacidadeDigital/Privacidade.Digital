Firefox: Configurações relacionadas à privacidade

Esta página é uma coleção de configurações about:config relacionadas à privacidade. Nós iremos te mostrar como fortalecer a privacidade do seu navegador Firefox.

Preparação:

  1. Digite "about:config" na barra de endereço do seu Firefox e aperto enter.
  2. Aperte o botão "Eu aceito o risco!" para continuar
  3. Siga as instruções abaixo...

Começando:

privacy.firstparty.isolate = true
Um resultado do projeto Tor Uplift, esta preferência isola todas as fontes de identificadores do navegador (exemplo: cookies) para o domínio principal, com o objetivo de previnir rastreamento entre domínios diferentes. (Não faça isso se estiver utilizando a extensão Firefox "Cookie AutoDelete" com Firefox versão 58 ou mais baixa.)
privacy.resistFingerprinting = true
Outro resultado do projeto Tor Uplift, esta preferência faz o Firefox ficar mais resistente à identificação da impressão digital do navegador.
privacy.trackingprotection.fingerprinting.enabled = true
Bloqueia identificação da impressão digital do navegador
privacy.trackingprotection.cryptomining.enabled = true
Bloqueia mineração de criptomoedas
privacy.trackingprotection.enabled = true
Essa é a proteção de rastreamento nova da Mozilla. Ela usa uma lista da Disconnect.me para filtrar rastreadores, o que é redundante se você já estiver usando uBlock Origin, portanto você deve deixá-la em false caso você esteja usando as funcionalidades dessa extensão.
browser.cache.offline.enable = false
Desativa cachê offline.
browser.safebrowsing.malware.enabled = false
Desativa as checagens de malware do Google. Risco de segurança, mas um ganho em privacidade.
browser.safebrowsing.phishing.enabled = false
Desativa a proteção de phishing do Google. Risco de segurança, mas um ganho em privacidade.
browser.send_pings = false
The attribute would be useful for letting websites track visitors' clicks.
browser.sessionstore.max_tabs_undo = 0
Mesmo que o Firefox esteja configurado para não armazenar o histórico, suas as abas fechadas são armazenadas temporariamente em Menu -> Biblioteca -> Histórico -> Abas fechadas recentemente.
browser.urlbar.speculativeConnect.enabled = false
Desative o pré-carregamento e auto-completar URLs. Firefox carrega URLs previamente que autocompletam quando um usuário digita na barra de endereço, o que é um problema se as URLs sugeridas não sejam de sites que o usuário queira se conectar. Fonte
dom.battery.enabled = false
O status da bateria do seu dispositivo poderia estar sendo rastreado.
dom.event.clipboardevents.enabled = false
Desative a funcionalidade que permite que sites recebam uma notificação quando você copia, cola our corte algo de uma página, e os deixa saber qual parte da página foi selecionada.
geo.enabled = false
Desative geolocalização.
media.eme.enabled = false

Desative a reprodução de conteúdo HTML5 controlados por DRM, que, se habilidada, automaticamente faz download do Módulo de Desencriptação Widevine distribuído pela Google Inc. Detalhes

Conteúdo controlado por DRM precisa que plugins Adobe Flash ou Microsoft Silverlight ainda sejam capazes de serem executados, instalados e habilitados no Firefox.

media.gmp-widevinecdm.enabled = false
Desativa o Módulo de Desencriptação Widevine distribuído pela Google the Widevine Content Decryption Module provided by Google Inc., usado para reproduzir conteúdo HTML5 controlado por DRM. Detalhes
media.navigator.enabled = false
Sites podem rastrear o status do microfone e câmera do seu dispositivo.
network.cookie.cookieBehavior = 1
Desativar cookies
  • 0 = Aceitar todos os cookies por padrão
  • 1 = Aceitar somente cookies do site atual (bloqueia cookies de terceiros)
  • 2 = Bloquar todos os cookies por padrão
network.cookie.lifetimePolicy = 2
cookies são deletados no fim da sessão
  • 0 = Aceitar cookies normalmente
  • 1 = Perguntar toda vez antes de aceitar um cookie
  • 2 = Aceitar somente para a sessão atual
  • 3 = Aceitar por N dias
network.http.referer.trimmingPolicy = 2
Enviar somente scheme, host, e porta no cabeçalho Referer
  • 0 = Enviar a URL completa no cabeçalho Referer
  • 1 = Enviar a URL sem a string query no cabeçalho Referer
  • 2 = Enviar somente scheme, host, e porta no cabeçalho Referer
network.http.referer.XOriginPolicy = 2
Enviar cabeçalho Referer somende quando os hostnames são exatamente iguais. (Nota: se você perceber que os sites não estiverem funcionando da forma correta, você pode tentar 1 combinado com a configuração XOriginTrimmingPolicy abaixo.) Fonte
  • 0 = Enviar cabeçalho Referer em todos os casos
  • 1 = Enviar cabeçalho Referer para os mesmos sites eTLD
  • 2 = Enviar cabeçalho Referer somente quando os hostnames são exatamente iguais
network.http.referer.XOriginTrimmingPolicy = 2
Quando enviando o cabeçalho Referer através de diferentes origens, enviar somente scheme, host, e porta no cabeçalho Referer de requisições com diferentes origens. Fonte
  • 0 = Enviar URL completa no cabeçalho Referer
  • 1 = Enviar URL sem a string query no cabeçalho Referer
  • 2 = Enviar somente scheme, host, e porta em cabeçalho Referer
webgl.disabled = true
Desabilitar WebGL, que pode ser um risco de segurança. Fonte
browser.sessionstore.privacy_level = 2
Essa preferência controla quando armazenar informação extra sobre a sessão: conteúdos de formulários, posições de barras de rolagem, cookies, e dados de POST. Detalhes
  • 0 = Armazenar dados extra da sessão para qualquer site. (Padrão a partir do Firefox 4.)
  • 1 = Armazenar dados extra da sessão somente para sites não encriptados (sem HTTPS). (Padrão antes do Firefox 4.)
  • 2 = Nunca armazenar dados extra da sessão.
extensions.pocket.enabled = false
Desabilitar completamente a extensão Pocket.
network.IDN_show_punycode = true
Not rendering IDNs as their Punycode equivalent leaves you open to phishing attacks that can be very difficult to notice. Source
extensions.blocklist.url = https://blocklists.settings.services.mozilla.com/v1/blocklist/3/%20/%20/

Limitar a quantidade de informação identificável enviada quando requisitando a lista de extensões perigosas da Mozilla.

Opcionalmente, a lista pode ser desabilitada completamente ao modificar extensions.blocklist.enabled para false para melhor privacidade, porém pior segurança. Fonte

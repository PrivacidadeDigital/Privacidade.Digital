Glossário

  • DNS-over-TLS (DoT) - Um protocolo de segurança para DNS encriptado, realizado na porta 853. Alguns provedores também suportam a porta 443, que geralmente funciona em qualquer local, enquanto a porta 853 é comumente bloqueada por firewalls restritivos. DoT tem dois modos:
    • Modo Oportunístico: o cliente tenta estabelecer uma conexão DNS-over-TLS com o servidor na porta 853, sem verificar a validade do certificado de segurança. Se a conexão falhar, DNS não-encriptado será utilizado.
    • Modo Estrito: o cliente se conecta com um hostname específico e verifica a validade do certificado de segurança do mesmo. Se a validação falha, nenhuma requisição DNS é feita até que a mesma ocorra com sucesso.
  • DNS-over-HTTPS (DoH) - Similar ao DoT, mas utilizando HTTPS, com tráfego indistinguível do tráfego HTTPS "normal" na porta 443.
  • DNSCrypt - Um método antigo de encriptar DNS, porém robusto.

Como Verificar Se o DNS Está Encriptado

  • DoH / DoT
    • Verifique no site DNSLeakTest.com.
    • Verifique no site do seu provedor DNS. Eles podem ter alguma página que verifica se você está utilizando o DNS deles. Exemplos incluem AdGuard e Cloudflare.
    • Se estiver utilizando o Trusted Recursive Resolver (TRR) do Firefox, navegue até about:networking#dns. Se a coluna TRR estiver como true para alguns campos, você está utilizando DoH.
  • dnscrypt-proxy - Verifique na wiki dnscrypt-proxy a guia de como verificar se sua DNS está encriptada (em Inglês).
  • DNSSEC - Verifique em DNSSEC Resolver Test por Matthäus Wander (em Inglês).
  • Minimização QNAME - Execute dig +short txt qnamemintest.internet.nl na linha de comando do seu sistema operacional (dica desse slide da NLnet Labs, em Inglês). Se você estiver utilizando Windows 10, execute Resolve-DnsName -Type TXT -Name qnamemintest.internet.nl no PowerShell. A seguinte mensagem deve ser retornada: "HOORAY - QNAME minimisation is enabled on your resolver :)!"

Menções Honrosas e Informações Adicionais

  • Clientes de DNS Encriptado para Computadores de Mesa e Laptops:
    • Firefox vem com suporte a DoH embutido com Cloudflare selecionado como resolvedor padrão, mas pode ser configurado para utilizar qualquer resolvedor DoH. Atualmente a Mozilla está conduzindo estudos antes de habilitar DoH por padrão para todos os usuários Firefox residentes dos EUA.
      • DNS-over-HTTPS pode ser habilitado em Menu -> Preferências (about:preferences) -> Configurações de rede -> Ativar DNS sobre HTTPS. Em "Usar provedor" selecione "Personalizado", e digite o endereço do seu provedor DoH.
      • Usuários avançados podem habilitar a configuração em about:config ao mudar os valores de network.trr.custom_uri e network.trr.uri para o endereço que você encontrar na documentação do seu provedor DoH e o valor de network.trr.mode para 2. Também pode ser vantajoso mudar o valor de network.security.esni.enabled para True, de forma que habilita SNI encriptado e faça sites que suportam ESNI um pouco mais difíceis de rastrear.

Definitions

DNS-over-TLS (DoT)

A security protocol for encrypted DNS on a dedicated port 853. Some providers support port 443 which generally works everywhere while port 853 is often blocked by restrictive firewalls.

DNS-over-HTTPS (DoH)

Similar to DoT, but uses HTTPS instead, being indistinguishable from "normal" HTTPS traffic on port 443 and more difficult to block. {% include badge.html color="warning" text="Warning" tooltip="DoH contains metadata such as user-agent (which may include system information) that is sent to the DNS server." link="https://tools.ietf.org/html/rfc8484#section-8.2" icon="fas fa-exclamation-triangle" %}

DNSCrypt

With an open specification, DNSCrypt is an older, yet robust method for encrypting DNS.

Anonymized DNSCrypt

A lightweight protocol that hides the client IP address by using pre-configured relays to forward encrypted DNS data. This is a relatively new protocol created in 2019 currently only supported by dnscrypt-proxy and a limited number of relays.

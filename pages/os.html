Lembre-se de checar por soluções de vulnerabilidades do CPU

Isso também afeta o Windows 10, mas essa solução não é exposta de forma muito acessível. Usuários de MacOS devem checar Como ativar a correção completa para as vulnerabilidades MDS (Microarchitectural Data Sampling).

Ao rodar uma versão recente do kernel Linux, você pode checar por vulnerabilidades do CPU que o mesmo detecta, utilizando o comando tail -n +1 /sys/devices/system/cpu/vulnerabilities/*. Ao utilizar tail -n +1 ao invés de cat, os nomes dos arquivos também são mostrados.

Caso você utilize uma CPU da Intel, pode ser que note a frase "SMT vulnerable" após rodar o comando mostrado anteriormente. Para consertar o problema, desabilite hyper-threading pela UEFI/BIOS. Você também pode tomar os seguintes passos caso seu sistema/distribuição utiliza GRUB e tem suporte a /etc/default/grub.d/:

  1. sudo mkdir /etc/default/grub.d/ para criar um diretório de configurações adicionais do grub
  2. echo GRUB_CMDLINE_LINUX_DEFAULT="$GRUB_CMDLINE_LINUX_DEFAULT l1tf=full,force mds=full,nosmt mitigations=auto,nosmt nosmt=force" | sudo tee /etc/default/grub.d/mitigations.cfg para criar um novo arquivo-fonte de configuração grub com o conteúdo gerado pelo comando echo
  3. sudo grub-mkconfig -o /boot/grub/grub.cfg para gerar o arquivo final de configuração grub
  4. sudo reboot para reiniciar
  5. após a reinicialização, checar tail -n +1 /sys/devices/system/cpu/vulnerabilities/* novamente para ver se tudo relacionado a SMT agora mostra "SMT disabled."
